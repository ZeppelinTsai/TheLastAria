# Asset Organization

This project keeps image assets under `img/`.

Use stable, readable English folder and file names. Prefer names that make sense to both humans and AI tools. Avoid temporary generated names such as `ChatGPT Image ...` once an asset becomes part of the game.

## Top-Level Image Folders

```text
img/
  bg/
  tachie/
  cg/
  sprite/
  prelude/
```

## Backgrounds

Use `img/bg/` for map and environment backgrounds.

Recommended structure:

```text
img/bg/<map_id>/<map_id>_base.png
img/bg/<map_id>/<map_id>_fog.png
img/bg/<map_id>/<map_id>_lights.png
img/bg/<map_id>/<map_id>_particles.png
```

For a simple single-layer background, this is also acceptable:

```text
img/bg/sunken_city.png
img/bg/sunken_city_lyra_room.png
```

When a map starts needing atmosphere layers, move it toward the folder form.

## Tachie

Use `img/tachie/` for dialogue standee art.

Recommended structure:

```text
img/tachie/<character>/<character>_<expression>.png
```

Examples:

```text
img/tachie/lyra/lyra_default.png
img/tachie/lyra/lyra_smile.png
img/tachie/lyra/lyra_sad.png
img/tachie/lumi/lumi_default.png
img/tachie/lumi/lumi_surprised.png
```

Use lowercase character names in paths. Use expression keys that match `data/characters.json`.

## CG

Use `img/cg/` for one-off event illustrations, storybook pages, memory scenes, endings, and cutscene stills.

Recommended structure:

```text
img/cg/<sequence_id>/<sequence_id>_<index>.png
```

Examples:

```text
img/cg/prelude_storybook/prelude_storybook_00.png
img/cg/prelude_storybook/prelude_storybook_01.png
img/cg/orion_memory/orion_memory_01.png
```

Current `img/prelude/` can remain for now, but future storybook or cutscene images should use `img/cg/`.

## Sprites

Use `img/sprite/` for in-world character sprites and animation frames.

Recommended structure:

```text
img/sprite/<character>/<state>/<animation>/<frame>.png
```

Examples:

```text
img/sprite/lyra/human/walk_down/lyra_human_walk_down_00.png
img/sprite/lyra/human/walk_down/lyra_human_walk_down_01.png
img/sprite/lyra/swim/idle/lyra_swim_idle_00.png
img/sprite/lyra/swim/up/lyra_swim_up_00.png
img/sprite/lumi/floating/idle/lumi_floating_idle_00.png
```

If a sprite sheet is used instead of frame images:

```text
img/sprite/lyra/human/lyra_human_sheet.png
img/sprite/lyra/swim/lyra_swim_sheet.png
```

## Naming Rules

- Use lowercase English names.
- Use underscores instead of spaces.
- Include character, state, expression, or map id in the filename.
- Avoid generated filenames after importing final assets.
- Keep `characters.json`, dialogue JSON, and scene paths updated when moving files.

## Migration Note

Do not bulk-move existing assets without updating:

- `data/characters.json`
- `data/maps/*.json`
- `data/dialogues/*.json`
- `scenes/**/*.tscn`
- matching `.import` files generated by Godot

Move one asset category at a time, then run `check_godot.bat`.
